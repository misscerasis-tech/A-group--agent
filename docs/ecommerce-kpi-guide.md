# 电商核心指标说明

这份文档解释 Agent 为什么优先看这些数据，以及首页为什么这样排序。原则是：先看结果，再找原因，再看风险，最后看外部压力。

## 首页排序

1. 销售额和订单数：先判断这周店铺整体变好还是变差。
2. 访客数、转化率和客单价：判断变化来自流量、购买意愿，还是商品结构。
3. 广告回本、毛利、库存和退款/退货：判断增长有没有代价、利润是否被吞掉、会不会断货或被售后吃回去。
4. 竞品价格、促销和卖点：判断外部环境有没有变，避免只从自己店里找原因。

## 指标解释

| 指标 | 小白理解 | 代表什么 | 首页体现 |
| --- | --- | --- | --- |
| 销售额 | 这周一共卖了多少钱 | 店铺整体结果 | 第一张指标卡 |
| 订单数 | 有多少人真的下单 | 需求是否变强或变弱 | 与销售额并列 |
| 访客数 / 曝光量 | 有多少人看到了或进了店 | 问题是否出在流量 | 缺失时 Agent 优先追问 |
| 转化率 | 进店的人里有多少买了 | 商品页、价格、优惠和评价是否有效 | 首页写成“进店后下单”；如果平台直接给转化率，Agent 可用它反推访客口径 |
| 客单价 | 每单平均花多少钱 | 套餐、折扣、高低价商品结构 | 作为二级诊断 |
| 广告花费 / 广告成交额 | 广告花了多少钱，带回多少成交 | 投放是否划算 | 首页写成“广告回本”；用“每花 1 个金额单位带回多少成交”解释，不硬猜币种 |
| 毛利 / 商品成本 / 平台履约费 | 卖完后真正留下多少钱 | 销售增长是否真的赚钱，是否被佣金、手续费、物流履约成本吃掉 | 低毛利商品进入风险诊断；如果有商品成本和平台/支付/履约费，会折进毛利口径 |
| 库存 / 可售天数 | 按现在速度还能卖几天 | 热卖断货和滞销压货风险 | 高风险 SKU 标成优先处理 |
| 退款 / 退货 | 买完后有多少人退掉 | 商品质量、物流和描述风险 | 首页、商品诊断和周报都能提示售后风险；如果有退款率或退款原因，会直接落到商品问题 |
| 竞品价格 / 促销 / 卖点 | 别人卖多少钱、怎么打折、主打什么理由 | 外部价格和购买理由压力 | 单独做竞品动态解释 |

## 平台只给“率”也能先跑

很多平台导出的不是完整原始数据，而是 `转化率`、`ROAS/投产比`、`毛利率`、`退款率` 这类比率。Agent 会优先使用原始字段；如果缺少原始字段但有比率，就反推一份可用于复盘的口径：

- 有 `订单数 + 客单价`，可以先补出销售额口径。
- 有 `订单数 + 转化率`，可以补出访客数口径。
- 有 `广告花费 + ROAS/投产比`，可以补出广告成交额口径。
- 有 `销售额 + 毛利率`，可以补出毛利口径。
- 有 `订单数 + 退款率`，可以补出退款/退货单数口径。
- 有 `销售额 + 退款金额占比`，可以补出退款金额口径。

如果没有毛利但有商品成本，Agent 会用 `销售额 - 商品成本 - 平台佣金 - 支付手续费 - 物流/履约成本 - 其他可变成本` 得到一个可用于判断的毛利口径。如果平台已经给了毛利，Agent 会优先使用平台毛利，不重复扣这些费用。

金额展示默认按原表金额单位理解，不自动把人民币、美元、欧元等互相换算，也不会在输出里硬加 `$` 或 `¥`。

## Agent 行为约束

- 缺流量数据时，只能说“不能判断是没人进店还是进店后不买”。
- 缺广告数据时，不能判断广告好坏。
- 缺成本或毛利时，不能把“卖得多”说成“赚得多”；只有手续费但没有商品成本时，也不能假装已经知道真实利润。
- 缺库存时，不能提前判断断货风险。
- 缺退款/退货数据时，不能判断售后是否在吃掉成交。
- 缺竞品时，只能给店铺内部复盘，不能假装知道外部价格变化。
