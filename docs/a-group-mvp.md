# A 组 MVP：小白电商运营 Agent

## 产品定位

这是一个飞书驱动的电商运营增长 Agent。它面向不懂专业电商运营的新手店主、创业团队和业务负责人，帮助用户完成每周店铺复盘、异常诊断、竞品观察和下周行动计划。

核心价值不是“展示很多图表”，而是“带着用户工作”：

- 用户给数据或授权平台。
- Agent 判断信息是否足够。
- Agent 主动追问缺口。
- Agent 自动分析店铺表现。
- Agent 用自然语言解释问题。
- Agent 输出可执行任务并回写飞书。

## A 组对应场景

主场景选择：电商运营场景。

覆盖工作流：

- 工作流一：电商数据周报 / 业绩复盘。
- 工作流二：竞品动态常态化监测与分析。

不把内容批量生产作为主线。内容生成只作为运营建议后的可选加分能力，例如“生成促销标题”。

## MVP 演示闭环

演示只跑一条完整链路：

1. 用户在飞书里说：“帮我看看这家店这周经营得怎么样。”
2. Agent 判断缺少店铺数据，主动询问要订单、商品、库存或广告数据。
3. 用户上传表格或选择样例数据。
4. Agent 自动识别字段，生成小白可读的数据摘要。
5. Agent 判断本周经营变好还是变差。
6. Agent 找到关键问题：流量、转化、客单价、广告、库存或竞品。
7. Agent 读取竞品链接或样例竞品数据。
8. Agent 生成“原因解释 + 下周 3 件事 + 风险提醒”。
9. Agent 回写飞书文档，并生成待办任务。

## Agent 输出风格

必须说人话，不把专业词扔给用户。

示例：

> 这周销售额下降了 18%。主要不是因为没人进店，而是进店后购买的人变少了。简单说，你的流量还在，但商品页说服力下降了。最可能的原因是竞品在打折，而你的主推款价格没有变化。建议先做三件事：调整主推款价格、暂停两个效果差的广告、检查库存只剩 8 天的 SKU。

## Agent 决策框架

Agent 每次分析按这 6 个问题工作：

1. 这周整体是变好还是变差？
2. 变化主要来自哪里？
3. 是流量问题、转化问题、价格问题、广告问题、库存问题，还是竞品问题？
4. 哪些商品该继续推？
5. 哪些商品该暂停、补货、降价或换图？
6. 下周最优先做哪 3 件事？

## MVP 功能清单

第一版必须有：

- 飞书任务入口或模拟飞书入口。
- 数据上传或样例数据选择。
- 字段识别和数据完整度检查。
- 小白版经营复盘。
- 商品级问题诊断。
- 库存风险提示。
- 竞品观察入口。
- 下周行动清单。
- 飞书结果回写或模拟回写。

第一版不做：

- 完整 BI 大盘。
- 多平台复杂 OAuth。
- 真实广告投放调参。
- 自动改价、自动上下架这类高风险动作。
- 大而全的内容生产中心。

## 比赛评分映射

业务价值：

- 降低新手做电商运营复盘的门槛。
- 帮助店主从“看不懂数据”变成“知道下一步做什么”。

自动化程度：

- 自动读取数据。
- 自动判断缺口。
- 自动诊断原因。
- 自动生成行动计划。
- 自动回写飞书。

AI 分析 / 生成质量：

- 不只总结数据，还要解释原因、判断优先级、生成任务。
- 输出必须自然语言、可执行、可被小白理解。

飞书集成体验：

- 飞书发起任务。
- 飞书接收结果。
- 飞书承载待办和审核。

可用性与稳定性：

- 允许用户数据不完整。
- 数据缺失时追问。
- 外部平台不可用时使用上传数据或样例数据降级。

## 推荐技术路线

原型阶段：

- 继续使用现有 Next.js 中文后台底座。
- 用本地样例数据和表格上传模拟真实平台接入。
- 用“模拟飞书会话”先跑通 Agent 工作流。

演示增强阶段：

- 接入一个真实外部平台或公开商品页。
- 接入飞书机器人 / 文档 / 多维表格中的至少一种。
- 保留样例数据作为现场稳定降级。

## 第一阶段交付物

- 用户准备清单。
- Agent 工作流设计。
- 样例经营数据结构。
- 可交互原型页面。
- 一条完整演示脚本。
