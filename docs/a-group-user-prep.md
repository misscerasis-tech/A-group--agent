# A 组用户准备清单

本项目定位为“小白也能用的电商运营 Agent”。用户不需要先理解专业运营术语，只要把能拿到的数据、链接和业务背景交给 Agent，Agent 负责追问缺口、解释结果并给出下一步行动。

## 你先准备什么

### 1. 店铺背景

最少回答这几个问题：

- 你卖什么产品？
- 主要在哪个平台卖？例如 Shopify、淘宝、京东、Amazon、抖音小店。
- 主要卖给谁？例如国内年轻女性、美国户外用户、巴西通勤人群。
- 你这周最关心什么？例如卖不动、广告太贵、库存不够、竞品降价。

如果暂时不知道，就直接写“不知道”，Agent 会按默认小白流程继续问。

### 2. 店铺经营数据

任选一种格式即可：

- Excel
- CSV
- 平台后台导出的订单表、商品表、库存表
- 截图
- 手动粘贴的一段数据

最有用的数据包括：

- 日期
- 商品名称或 SKU
- 销售额
- 订单数
- 访客数或曝光量
- 加购数
- 支付转化率
- 广告花费
- 广告成交额
- 库存数量

不需要一次全有。第一版 Agent 会先判断数据完整度，再告诉用户缺什么。

### 3. 广告或流量数据

有就给，没有也可以先跳过。

常见来源：

- Google Ads
- Meta Ads
- 巨量引擎
- 平台站内广告
- Google Analytics / GA4

Agent 应该用人话解释广告效果，例如“花的钱有没有带来订单”，而不是只展示 ROAS、CPC、CTR 这类缩写。

### 4. 竞品信息

最少给 1 到 3 个竞品链接或竞品名称。

可以是：

- 商品详情页链接
- 店铺链接
- Amazon 商品链接
- 淘宝/京东商品链接
- 小红书、抖音、TikTok、Instagram 上的竞品内容链接

如果没有竞品，Agent 会先用用户产品关键词生成待观察竞品清单。

### 5. 飞书条件

比赛要求飞书接入。优先准备：

- 一个飞书群
- 一个飞书应用或机器人
- 一个飞书文档或多维表格用于接收结果

如果飞书应用权限暂时没好，原型阶段先做“模拟飞书入口”，但最终演示必须展示真实飞书入口、结果回写和任务流转。

## 用户不需要准备什么

- 不需要会看运营指标。
- 不需要会写 Prompt。
- 不需要提前整理成固定模板。
- 不需要自己判断问题原因。
- 不需要自己写周报。

这些都是 Agent 的工作。

## Agent 应该主动问的问题

当信息不足时，Agent 应该按优先级追问：

1. 这家店卖什么，主要在哪个平台卖？
2. 你希望我先看这周、这个月，还是最近 7 天？
3. 你有没有订单、商品、库存或广告数据？
4. 你最担心的是销量、利润、广告成本、库存还是竞品？
5. 有没有你想对比的竞品链接？
6. 结果要发给谁，是否需要生成飞书待办？

## 第一轮可以直接给我的最小包

如果只想快速启动，给下面这些就够：

- 1 个产品名称。
- 1 份订单或销售表。
- 1 份库存表，没有也可以。
- 1 到 3 个竞品链接，没有也可以。
- 你希望 Agent 用飞书群、文档还是多维表格承载结果。
