# A 组现场演示脚本

## 作品一句话

这是一个飞书驱动的小白电商运营 Agent。用户不用懂电商指标，只要给店铺数据或平台授权，Agent 会主动追问信息、分析经营问题、解释原因，并把下周行动计划回写飞书。

## 角色设定

演示用户是一个新手店主：

- 卖智能温显保温杯。
- 用 Shopify 做独立站。
- 最近一周感觉销量不稳定。
- 不知道问题是广告、库存、价格、流量还是竞品。

Agent 是电商运营教练：

- 不假设用户懂专业术语。
- 信息不足时先问问题。
- 有数据后自动分析。
- 输出结果必须能直接照着做。

## 演示主线

### 1. 飞书发起任务

用户在飞书说：

> 帮我看看这家店这周经营得怎么样。

讲解重点：

- 这不是普通聊天，而是一个业务流程入口。
- Agent 会先判断任务需要什么信息。

### 2. Agent 主动追问

Agent 问：

- 你卖什么产品？
- 主要在哪个平台卖？
- 有没有订单、库存、广告或流量数据？
- 有没有竞品链接？
- 这周更想保销量还是保利润？

讲解重点：

- 面向小白，不要求用户先懂运营模型。
- Agent 主动把模糊需求拆成可工作的输入。

### 3. 用户给数据

原型阶段使用样例数据，真实阶段可替换为：

- Shopify 订单导出。
- 平台商品表。
- 库存表。
- Google Analytics 或广告数据。
- 客服备注、商品评价或退款备注。
- 竞品链接。

讲解重点：

- Agent 会做数据完整度检查。
- 缺流量、广告、库存时不会强行胡说，而是告诉用户缺什么。
- 如果有客服备注或差评关键词，Agent 会用这些用户声音解释退款/退货背后的具体原因。

### 4. Agent 生成小白版经营复盘

打开 `/agent` 页面展示：

- 本周销售额和订单变化。
- 进店后下单比例变化。
- 广告回本变化。
- 商品级问题诊断。
- 竞品价格和促销观察。

讲解重点：

- 不只展示图表，而是把数据翻译成自然语言。
- 比如“进店的人还在，但买的人变少了”，比“转化率下降”更适合新手。

### 5. Agent 给出下周行动清单

展示：

- 先处理主推款购买理由。
- 暂停最弱广告。
- 检查礼盒套装补货或控量。
- 建立每周飞书复盘节奏。

讲解重点：

- Agent 不止分析，还要带用户继续工作。
- 每个动作都有负责人、原因和第一步。

### 6. 回写飞书

展示“飞书回写预览”：

- 经营结论。
- 下周待办。
- 风险提醒。

讲解重点：

- 后续接真实飞书机器人后，结果自动发群或写入文档。
- 飞书承担任务入口、结果接收和协同流转。

## 评委可能问什么

### Q1：这和普通 BI 看板有什么不同？

回答：

普通看板要求人自己懂指标、自己找原因。我们的 Agent 先追问、再分析、再解释、再安排任务，目标是让不懂运营的人也能完成一次经营复盘。

### Q2：如果数据不完整怎么办？

回答：

Agent 不会硬编结论。它会标记数据缺口，例如缺流量就不判断转化率，缺广告就不判断广告回本，然后主动问用户补哪份数据。

### Q3：外部平台怎么接？

回答：

第一阶段支持上传表格和样例数据保证稳定演示；正式接入时优先接飞书和一个外部平台，例如 Shopify、Google Analytics 或公开竞品商品页。

### Q4：怎么体现业务价值？

回答：

它把每周复盘从“人手动拉表、算指标、写报告、派任务”变成“飞书一句话触发，Agent 自动拉数据、解释问题、生成行动清单并回写飞书”。
